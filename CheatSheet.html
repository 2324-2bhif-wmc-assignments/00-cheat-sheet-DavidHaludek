Word Wide Web General

1. Die entscheidenden Technologien, die das World Wide Web so erfolgreich gemacht haben, sind das Hypertext Transfer Protocol (HTTP), die Uniform Resource Identifiers (URIs) zur Adressierung von Ressourcen, und die Hypertext Markup Language (HTML) zur Strukturierung von Inhalten.

2. Ressourcen wie HTML-Seiten und Bilder werden im Web durch eindeutige Adressen, sogenannte Uniform Resource Locators (URLs), identifiziert.

3. Eine URL besteht aus dem Protokoll (z.B. "http" oder "https"), dem Hostnamen (z.B. "www.example.com") und dem Pfad zur spezifischen Ressource (z.B. "/pfad/zur/ressource").

Web Server

1. Ein Webserver ist eine Software oder Hardware, die HTTP-Anfragen von Clients entgegennimmt und entsprechende Ressourcen zurücksendet.

2. Ein Webserver liefert Webinhalte wie HTML-Seiten, Bilder und andere Dateien an Clients aus.

3. Inhalte können auf einem Webserver gespeichert werden, indem sie auf dessen Dateisystem hochgeladen oder über eine Datenbank verwaltet werden.

4. Der Inhalt einer Webseite wird mithilfe von HTML (Hypertext Markup Language) beschrieben, die die Struktur und das Layout definiert.

Web Client

1. Ein Webclient ist eine Software, die in der Lage ist, HTTP-Anfragen an Webserver zu senden und die empfangenen Ressourcen anzuzeigen.

2. Ein Webclient kann Webseiten anzeigen, Formulare ausfüllen, Daten senden und empfangen, etc.

Nein, ein Webclient ist eine allgemeine Bezeichnung für eine Software, die mit Webservern kommunizieren kann. Ein Webbrowser ist eine spezifische Art von Webclient, der in der Lage ist, Webseiten anzuzeigen.
